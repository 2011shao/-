# 部署与回滚手册 v1.0

## 1. 环境

- dev / test / uat / prod
- 统一使用容器部署（建议 Docker + CI/CD）

## 2. 部署步骤

1. 拉取发布分支代码
2. 构建后端镜像与前端静态资源
3. 执行数据库迁移脚本
4. 发布应用
5. 执行健康检查

## 3. 健康检查

- `/health` 返回 200
- 数据库连接正常
- Redis（如使用）连接正常
- 登录与核心接口可访问

## 4. 回滚步骤

1. 切回上一个稳定镜像
2. 若涉及不可逆 DB 变更，执行回滚脚本或恢复备份
3. 重启服务并复检核心链路

## 5. 监控建议

- API 错误率
- P95 延时
- DB 慢查询
- 应用 CPU/内存
- 审计日志异常峰值
