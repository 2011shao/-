# 线下手机门店 ERP 文档中心（一期）

> 范围：多店铺、供应商、品牌与二级分类、入库与库存、串码全局唯一与可追溯。
>
> 不包含：财务系统、移动端。

## 文档目录

- `prd/PRD_v1.md`：产品需求说明书（范围、流程、权限、验收标准）
- `api/API_SPEC_v1.md`：接口规范与关键 API 清单
- `db/DATA_DICTIONARY_v1.md`：数据字典（字段、约束、索引、枚举）
- `db/ERD_v1.md`：ER 关系与建模说明
- `testcase/TEST_CASES_v1.md`：测试计划与核心测试用例
- `test/TEST_RUN_GUIDE_v1.md`：测试运行文档（环境、冒烟、回归）
- `release/GO_LIVE_CHECKLIST_v1.md`：上线检查清单
- `ops/DEPLOY_RUNBOOK_v1.md`：部署与回滚手册

## 版本约定

- 当前版本：`v1.0`
- 版本策略：需求、接口、数据结构发生变更时，必须更新对应文档版本与变更记录。

## 变更流程（强制）

1. 提出变更（需求单/缺陷单）
2. 评审通过（产品 + 技术 + 测试）
3. 更新文档（PRD/API/DB/TEST）
4. 开发实施
5. 回归验收
6. 发布上线

## 角色协同

- 产品：维护 PRD 与业务规则
- 前端：遵循 API 与页面字段规范
- 后端：遵循数据字典、权限、事务与审计规范
- 测试：按测试用例执行并回填结果
- 运维：按部署手册与上线清单执行
