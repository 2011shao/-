# PRD v1.0 - 多店铺线下手机门店 ERP（一期）

## 1. 项目目标

建设一个 PC 端 ERP，支撑多店铺运营，完成供应商、品牌与二级分类、入库、库存、串码追溯、调拨盘点、轻量出库管理。

### 1.1 成功标准

- 串码全局唯一，不允许重复入库。
- 任意串码可查询完整流转轨迹。
- 总店与分店数据权限清晰，无越权。
- 核心流程稳定：入库、调拨、盘点、出库。

## 2. 范围定义

### 2.1 In Scope（一期包含）

1. 基础资料
   - 品牌管理
   - 分类管理（严格二级）
   - 供应商管理
   - 门店与仓库管理
2. 入库管理
   - 入库单创建、审核、作废
   - 串码手动录入/批量导入/自动生成
3. 库存管理
   - 实时库存
   - 库存流水
   - 调拨、盘点、报损报废
4. 串码中心
   - 串码全局唯一校验
   - 串码状态与轨迹日志
5. 轻量出库（非财务）
6. RBAC 权限与审计日志

### 2.2 Out of Scope（一期不包含）

- 财务（应收应付/总账/利润核算）
- 移动端（小程序/APP）
- 商品主数据管理（SPU/SKU 主档）

## 3. 核心业务规则

1. **不建设商品管理模块**：入库明细以“业务快照”方式记录（品牌、二级分类、规格、进价、售价）。
2. **分类严格二级**：一级分类 -> 二级分类，不允许三级。
3. **门店入库权限**：
   - 总店：可选择任意其他店铺/仓库入库。
   - 其他店铺：仅可入库到本店仓库。
4. 手机类强制串码管理。
5. 串码支持：手动录入、导入、按数量自动生成。
6. 串码全局唯一（服务层 + DB 唯一索引双重保障）。
7. 串码全链路留痕（操作时间、操作人、来源单据、动作、仓库变化）。

## 4. 角色与权限

- 总部管理员：全量可见，可跨店操作
- 总店仓管：可对所有门店进行入库指定
- 分店店长：本店全量权限
- 分店仓管：本店库存操作权限
- 审计角色：只读 + 日志导出

## 5. 页面信息架构（PC）

1. 工作台
2. 基础资料
   - 品牌管理
   - 分类管理（二级）
   - 供应商管理
   - 门店/仓库管理
3. 入库管理
   - 入库开单
   - 入库单列表
4. 库存管理
   - 库存查询
   - 串码查询
   - 调拨管理
   - 盘点管理
   - 报损报废
5. 出库管理
6. 报表中心
7. 系统管理（用户/角色/日志）

## 6. 关键流程（简版）

### 6.1 入库流程

1. 选择目标门店/仓库（按权限控制可选范围）
2. 选择供应商
3. 录入明细：品牌、分类、规格、进价、售价、数量
4. 录入串码（手动/导入/自动）
5. 校验通过后提交审核
6. 审核入库 -> 更新库存 -> 写串码日志

### 6.2 串码追溯流程

输入串码 -> 查询主档 -> 展示事件时间线（入库、调拨、盘点、出库、售后）

## 7. 验收标准（UAT）

1. 重复串码无法提交入库。
2. 总店可选择其他门店入库，分店不可以。
3. 入库后库存与串码状态一致。
4. 串码时间线完整无断点。
5. 核心接口与页面满足性能要求（列表接口 P95 < 500ms，单据提交 P95 < 800ms）。

## 8. 里程碑（建议）

- W1-W2：需求/原型/ER 评审
- W3-W4：基础资料 + 权限
- W5-W6：入库 + 串码中心
- W7-W8：库存/调拨/盘点
- W9：出库 + 报表
- W10：UAT + 上线

## 9. 风险与对策

1. 串码并发冲突：DB 唯一索引 + 重试机制
2. 数据越权：后端强制门店过滤，不信任前端参数
3. 脏数据：规格录入规范 + 可选快捷模板
4. 发布风险：灰度发布 + 可回滚脚本
